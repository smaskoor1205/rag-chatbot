# Frontend Skill

- React architecture: feature-oriented components, route-level screens, typed API clients, and clear data ownership.
- Component design: small accessible primitives composed into application workflows.
- Hooks: isolate stateful behavior in reusable hooks with typed inputs and outputs.
- State management: prefer server-backed state, local reducer state for chat flows, and explicit loading/error states.
- Tailwind standards: semantic utility composition, responsive constraints, focus states, and consistent spacing.
- Accessibility: keyboard navigation, visible focus, form labels, ARIA only where native semantics are insufficient.
- Testing: unit tests for hooks and components, integration tests for authentication and chat workflows.
